Hydraulisc is now mostly Open Source!

Important update (2025-10-21):

This blog entry is considered stale. It has been reuploaded with the blog reworks for archiving purposes.

As of this update, certain... policies have been made, certain goals of ours have changed, and our general vision for The Hydraulisc Network has changed slightly.

We now aim to foster a free as in price, free as in freedom, open source, privacy respecting, alternative to most major social networks.

Your data should be yours. Even when you explicitly allow third parties to store certain parts of it.


Also, Hydraulisc has since been rebuilt, fully FOSS. Repository found at GitHub/Foss-Hydraulisc. A vague rundown on that decision: blog article from 2025-01-03. A vague, and unfollowed, roadmap haulisc-roadmap. Self-Hosting documentation: Hydraulisc Documentation.

You read that right.

Hydraulisc's frontend code is now mostly Free and Open Source! Along with this change, we are working on guides to make self-hosting as easy as one, two, host!


This change comes after we decided to make the build stack trace from two repositories; in other words, we now have our closed-source repo with only the server code, whilst the Open Source frontend is our actual frontend.

The new stack trace is as follows: "rm -rf ./* ./.??* && git clone https://github.com/Hydraulisc/hydraulisc-app . && git clone https://github.com/Hydraulisc/hydraulisc . && npm i && npm run build && npm run live".


Explanation

What does all that do?

In order:

  • "rm -rf ./* ./.??*"

    • Clears the current folder (and hidden files, such as .git or .env)

  • "git clone Hydraulisc/hydraulisc-app ."

    • Clones the private server source code into the current directory

  • "git clone Hydraulisc/hydraulisc ."

    • Clones the open source frontend code into the current directory

  • And finally, "npm i && npm run build && npm run live"

    • Installs dependencies, runs the build script to build some custom dependencies and runs the server on port 5500 on my server.
